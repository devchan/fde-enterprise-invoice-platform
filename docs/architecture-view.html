9.0

  • Refresh-token rotation; replayed refresh tokens rejected
  • Server-enforced tenant isolation & RBAC on every privileged action

  • Distributed rate limiting (login + upload) via Redis sliding window

Residual & accepted risks

  • CSRF: mitigated by SameSite=Lax cookies; dedicated double-submit CSRF tokens remain as defense-in-depth.
  • Prompt injection via invoice content: blast radius bounded by strict JSON schema + extra=forbid; few-shot examples come only from the org's own approved invoices; NL search produces filter JSON, never SQL. No explicit content sanitization yet.
  • Auto-approval risk: gated on full validation + anomaly pass + overall and per-field confidence (0.92 bar), one-flag kill-switch (AUTO_APPROVAL_ENABLED), distinct audit action — but thresholds are not yet tuned on production invoice data.
  • Agent surfaces: MCP + assistant share one tool layer that inherits tenant/RBAC scoping; the assistant is read-only by design. MCP runs stdio-only today — remote MCP would need per-client auth first.
  • Secrets management: env / K8s Secret today — no external vault or automated rotation.
  • Edge protection: no WAF / bot management in front of the ingress yet.
