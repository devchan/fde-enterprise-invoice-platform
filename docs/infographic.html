The processing pipeline

A real ordered flow — each invoice moves through nine stages, from upload to a logged decision. Uncertain fields divert to human review; confident ones can auto-approve untouched.

01Authenticated loginHS256 bearer tokens; every request scoped to one tenant with role enforcement.RBAC · Multi-tenant
02Secure uploadPDF / JPEG / PNG only, rate-limited, SHA-256 checksum with a duplicate-upload guard.Rate-limited · Deduped
03Store the fileLocal or S3-compatible storage, org/invoice-scoped keys, short-lived signed download URLs.S3 · Signed URLs
04Queue the jobA processing job is created at upload time and handed to a Redis-backed worker.Redis queue
05AI extractionOpenAI Responses API returns strict invoice JSON + line items, with per-field confidence and prompt versioning.Structured output
06Validate & enrichRule engine, anomaly detection (amount z-score + near-duplicate embeddings), plain-language fix suggestions.Rules · Anomalies
07Human reviewReviewer cockpit surfaces weak fields, similar invoices, and an AP assistant — correct and confirm.Cockpit · Assistant
08Approve or rejectA confidence-gated path auto-approves clean invoices; the rest get a human verdict.Touchless-capable
09Audit logEvery upload, correction, and decision is written to an append-only, tenant-scoped audit trail.Append-only
ApprovedPassed validation or a reviewer sign-off — recorded with actor, timestamp, and reason.
RejectedSent back with a decision event; failed jobs can be inspected and manually reprocessed.

03

Where the AI earns its keep

Not just extraction — retrieval, similarity, anomaly detection, and two agent surfaces that share one tenant-safe tool layer.

Structured extraction
Strict JSON schema with per-field confidence and persisted prompt versions; image downscaling trims provider cost.
Retrieval-augmented
Recent approved invoices from the same supplier are injected as few-shot examples to sharpen extraction.
Vector similarity
pgvector cosine distance over an HNSW index powers tenant-scoped similar-invoice and near-duplicate detection.
Anomaly detection
Supplier amount outliers via z-score over approved history, plus embedding near-duplicates, demote invoices back to review.
Touchless approval
Confidence-gated auto-approval with its own audit action and Prometheus counter — clean invoices never touch a human.
Model tiering
A cheap model runs first and escalates only on low confidence, with aggregated cost accounting per invoice.
Natural-language search
Queries translate to tenant-scoped filters, with a deterministic no-key fallback parser for offline dev.
AP assistant agent
A read-only tool-calling agent answers operational questions with a full tool trace — "why is this invoice stuck?"
MCP server
Search, detail, similarity, audit, accuracy, and reprocess tools exposed to any MCP client as a scoped service user.
